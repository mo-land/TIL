## 第3章 基本的なコマンドを覚えよう
### 1. コマンド操作の基本
- Tabキー2回
  補完候補を全て表示する
- Ctrl + R （インクリメンタル検索）
  コマンド履歴を検索できる。
  1文字入力するごとにその時点での候補が表示される。
  途中終了は Ctrl + C
#### 1-1. コマンド履歴について
- デフォルトで1000行保存される。増やしたい場合は、環境変数で設定を変更する・
- コマンド履歴は、シェルを終了する時点でホームディレクトリ内の.bash_history ファイルに保存される。
#### 1-2. パイプ（`|`）とリダイレクト（`>` `>>`）
|コマンド|説明|
|-|-|
| `コマンド1 \| コマンド2` |コマンドの出力を別のコマンドへと渡して処理させることができる。|
|`ls /etc`|	/etcディレクトリ内のファイル一覧を表示する|
|`ls /etc \| wc -l`|/etcディレクトリ内のファイル数を数える|
|`ls -l /etc \| less`|/etcディレクトリ内のファイル一覧をlessコマンドで表示|
|`コマンドA > 出力先ファイル名`|【上書き】コマンドAの実行結果を出力先ファイル名のファイルに保存|
|`コマンドA >> 出力先ファイル名`|【末尾に追記】コマンドAの実行結果を出力先ファイル名のファイルに保存|
|`ls /etc >> filelists`|【上書き】/etcディレクトリ内のファイル一覧をfilelistsというファイルに保存|

#### 1-3. メタキャラクタ
- メタキャラクタ
  特殊な意味を持つ記号
- ワイルドカード
  メタキャラクタの1種。ファイル名のパターンを表す特殊な記号
  |メタキャラクタ|意味|
  |-|-|
  |`*`|0文字以上の任意の文字列|
  |`?`|任意の1文字（文字数を限定したいときに使う）|

### 2. 環境変数
#### 2-1. 主な環境変数
|環境変数|説明|
|-|-|
|PATH|コマンドやプログラムを検索するディレクトリのリスト|
|USER|現在のユーザー名|
|LANG|地域設定（言語処理方式）|
|HOME|現在のユーザーのホームディレクトリ|
|HISTSIZE|コマンド履歴の最大値|
|HISTFILE|コマンド履歴を格納するファイル|
|EDITOR|標準エディタ|

|コマンド|意味|
|-|-|
|`echo $環境変数名`|【$つける！】環境変数の値を表示する|
|`export 環境変数名`|【$つけない！】環境変数の値を変更する|

※`export 環境変数名` で設定した値は、シェルが終了するまで有効。
  次回ログイン後も設定を有効にする場合は、`export HISRSIZE=2000`のような設定をホームディレクトリにある「.bash_profile」というファイルの末尾に追加する。

### 3.ファイルとディレクトリの操作
#### 3-1．Linuxのファイル
ファイル名は大文字と小文字が区別される。
`.`で始まるファイルやディレクトリは隠しファイル（隠しディレクトリ）となる。

ファイル・ディレクトリの主な種類は以下4つ。詳細は書籍にて。

- 通常ファイル
- ディレクトリ
- リンクファイル
- 特殊ファイル（Windowsにはない、デバイスファイルなどを含む）

#### 3-2．ディレクトリの構造
- `/` →　ルートディレクトリを表す！
- `pwd` → カレントディレクトリを確認できるコマンド
- ホームディレクトリ
  → ユーザーがログインしたときにカレントディレクトリとなるディレクトリ
