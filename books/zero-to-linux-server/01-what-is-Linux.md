## 第1章 Linuxって何
### 01_LinuxとはどのようなOSか
#### カーネルとディストリビューション
- カーネル
  OSの中核となるプログラム。「Linux」は、カーネルに付けられた名称。
- ディストリビューション
  カーネルに、日常的に使うために必要なオープンソースソフトウェア群を組み合わせてインストーラーと一緒に配布したもの。一般的に「Linux OS」はディストリビューションを指す。
#### ディストリビューションの種類
- Red Hat系ディストリビューション
  - 米Red Hat社が開発
  - 企業向け
  - 長期のサポートを約束
  - バージョンアップは数年に一度（ゆったりペース）
- Devian系ディストリビューション
  - Ubuntuはこっち寄り。
  - Ubuntuは、半年に1度最新版がリリースされる
- その他ディストリビューションもある

### 02_Linuxとソフトウェア
#### ディストリビューションの構成
- シェル
- コマンド
- 各種ライブラリ（プログラムの共通部品）
- GUI
- クライアントソフトウェア
#### 主なサーバーソフトウェア
Linuxディストリビューションには、様々なサーバーソフトウェアが標準で用意されている。
以下はその一例。
- Apache HTTP Server（Apache(ｱﾊﾟｯﾁ)と呼ばれることも多い）
  → シェアNo1
- nginx（エンジンエックス と読む）
  →パフォーマンスが良く、大量の処理も軽快にこなす。
  リバースプロキシ（大量アクセスをWebサーバーに代わって受け付けるサーバー）としても使われる
- Postfix（ポストフィックス と読む）
  → Linuxで標準的に使われているメールサーバー
- Dovecot（ダブコット）
  → POP・IMAPに対応したメールサーバー
- BIND（バインド）
  最も広く使われてるDNSサーバー
  （DNS ＝ ホスト名・ドメイン名とIPアドレスの対応付けを行うシステム）
- Samba（サンバ！）
  これを導入すると、LinuxサーバーをするWindowsサーバーの代替として使える
- Squid（スキッド）
  もっとも有名なプロキシサーバー
  （プロキシサーバー ＝ 社内→社外へWebアクセスする時、社内のクライアントに代わってWebサーバへ代理でアクセスするサーバー）
