## CI/CD漫才「せっかちすぎる開発現場」
登場人物  
CI：確認魔。すぐテストしたがる  
CD：出たがり。すぐデプロイしたがる  
***
CI「どうもー！CIです！」  
CD「CDです！二人合わせて継続的なんとかです！」  
  
CI「雑やなあ。せめて Continuous くらい言うて」  
CD「横文字多いとお客さん帰るから」  
  
CI「帰らへんわ。で、僕らの役割説明しよか」  
CD「はい。CIは、コードが変わるたびに“これ大丈夫？”って確認する係です」  
CI「そうそう。ビルドして、テストして、壊れてへんか見る」  
CD「で、私は“いけるな”思ったら即世に出す係です」  
CI「こわっ。言い方が世に放つ犯罪者やねん」  
  
CD「スピード感が大事やから」  
CI「いや大事やけど、確認なしで出したら事故るやろ」  
CD「でも確認って、だいたい長いねん」  
CI「必要やねん！」  
CD「お前、毎回“単体テストよし、結合テストよし、静的解析よし、カバレッジよし…”って点呼長すぎるねん」  
CI「飛行機飛ばす前くらい慎重でええやろ！」  
  
CD「私は違う。合格したらすぐ出す。迷わない」  
CI「そこも極端やねん。お前この前なんて言うた？」  
CD「“main に入った？ ほな本番や”」  
CI「居酒屋のノリで本番出すな！」  
  
CD「だって自動化したいし」  
CI「自動化はええ。でも安全装置は要る」  
CD「付いてるやん。お前が」  
CI「急に信頼してくるやん」  
  
CD「正直、私はCIのこと最初うっとうしい思ってた」  
CI「ほう」  
CD「なんでもかんでも止めよるから。“テスト落ちました”“Lintエラーです”“依存関係が…”うるさいねん」  
CI「事実を報告してるだけや！」  
CD「こっちからしたら、ライブ会場行こうとしてるのに玄関でずっと持ち物チェックされてる感じや」  
CI「パスポート忘れて海外行くタイプやなお前」  
  
CD「でも最近わかった」  
CI「何が？」  
CD「お前が雑なコード止めてくれるから、私が堂々と出られる」  
CI「……ええこと言うやん」  
CD「“テスト済みです”って顔で本番行ける」  
CI「顔で本番行くな、証跡で行け」  
  
CD「逆にお前も、私おらんかったらどうなん？」  
CI「え？」  
CD「ずっと確認して終わりやろ？」  
CI「……まあ、そうかも」  
CD「テスト緑でも、誰にも届かんかったら自己満足やで」  
CI「刺さるなあ」  
  
CD「CIは“ちゃんと作れてるか”を見る」  
CI「CDは“ちゃんと届ける”を担う」  
CD「せやから二人で一組や」  
CI「ええ話っぽくまとめに来たな」  
CD「CIだけでもあかん、CDだけでも危ない」  
CI「うん」  
CD「つまり僕らは…」  
CI「うん」  
CD「確認してから、ええ感じに出すコンビです！」  
CI「説明がまだちょっと雑やな！」  
  
二人「どうも、ありがとうございましたー！」  
  

## CI/CD漫才「GitHub Actionsで揉める二人」
登場人物  
CI：GitHub Actionsが回るたびに張り切る確認担当  
CD：デプロイしたくてたまらない公開担当  
***
CI「どうもー、CIです！」  
CD「CDです！ 本日は GitHub Actions の workflow に乗ってやってまいりました！」  
  
CI「登場から自動実行されとるやん」  
CD「on: push で来てます」  
CI「軽いなあ」  
  
CD「今日はわたしたち CI/CD の違いを、技術寄りに説明したいと思います」  
CI「急にまじめやな」  
CD「“CIって何？” “CDって何？” “どっちもGitHub Actionsでしょ？”って雑にまとめられがちなんで」  
CI「それはある」  
  
CI「まず僕、CIはコードをマージしたり push したりしたときに、壊れてないか自動で確認する係です」  
CD「はいはい。お前すぐ走るもんな」  
CI「git push された瞬間にね」  
CD「通知くるもんな。“workflow started”って」  
CI「で、まずソース取ってきて」  
CD「actions/checkout やな」  
CI「そう。で、言語の実行環境入れて」  
CD「Ruby とか Node.js とか」  
CI「依存関係インストールして」  
CD「bundle install とか npm ci とか」  
CI「そのあとテスト！」  
CD「出た、テスト大好きおじさん」  
  
CI「RSpec回します、Jest回します、Lint見ます、型チェックします、必要ならビルドもします」  
CD「お前、盛りだくさん定食やな」  
CI「壊れたコードを main に入れないためや！」  
CD「でもお前、よう止めるやん」  
CI「止めるで。1個でも失敗したら赤や」  
CD「怖いわあ。“1 example, 1 failure” で人格否定みたいな空気出すもんな」  
CI「RSpecは人格否定してへん！」  
  
CD「で、私CDはその先。確認の通った成果物を、実際の環境に届ける係です」  
CI「ここ大事やな」  
CD「はい。CIが“ちゃんと動きそう”を見て、CDが“実際に使える状態にする”」  
CI「たとえば staging にデプロイするとか、本番にデプロイするとか」  
CD「そうそう。GitHub Actions で言うたら、テスト成功後に deploy job が走る感じ」  
CI「needs: test って書かれてるやつやな」  
CD「そう。私はCIの合格通知がないと前に出られへん」  
CI「意外と礼儀正しい」  
  
CD「ただし、私も種類あるからな」  
CI「お、出た」  
CD「Continuous Delivery と Continuous Deployment」  
CI「名前似すぎ問題」  
CD「Delivery は“いつでも出せる状態まで自動で持っていく”」  
CI「最後の本番反映は人がボタン押す場合もある」  
CD「Deployment は“そこも含めて自動で本番まで出す”」  
CI「テスト通ったらそのまま本番、みたいな」  
CD「せや」  
CI「初学者のころ、ここごっちゃになるよな」  
CD「“CDってデプロイのD？”って思ったら半分合ってて半分ややこしいやつ」  
  
CI「でもお前、すぐデプロイしたがるから危ないねん」  
CD「いやいや、ちゃんと条件付きやで」  
CI「ほんまか？」  
CD「たとえば if: github.ref == 'refs/heads/main' とか」  
CI「main限定」  
CD「あと secrets 使ってクラウドにログインして」  
CI「secrets.AWS_ACCESS_KEY_ID とか RAILS_MASTER_KEY とか」  
CD「で、サーバーやホスティング先に反映していく」  
CI「Render、Vercel、AWS、Fly.io、そのへんやな」  
CD「そう。アプリを世に出す花形や」  
  
CI「花形ぶるな。お前、前に secrets 足りなくてコケてたやろ」  
CD「あれは環境側の事情です」  
CI「ログにしっかり出てたで。“missing secret”って」  
CD「GitHub Actions のログ、容赦ないねん」  
CI「お前が本番行く前にバレてよかったやろ」  
CD「それはまあ、そう」  
  
CD「逆にお前もあるやろ」  
CI「何が」  
CD「テストは全部通ったのに、デプロイしたら落ちるやつ」  
CI「……ある」  
CD「“ローカルでは動いてました”ってやつ」  
CI「やめろ」  
CD「migration 漏れとか、環境変数未設定とか、外部サービスとの接続とか」  
CI「胃が痛くなるラインナップやな」  
CD「せやから staging で確認したり、health check 入れたり、ロールバック考えたりするわけや」  
CI「CDも思ったより繊細なんよな」  
  
CD「そうやで。私はただの“公開ボタン連打マン”ちゃう」  
CI「まあ、見た目はだいぶそれやけどな」  
CD「失礼やな！」  
  
CI「じゃあ整理しよか」  
CD「おう」  
CI「CIは、push や pull request をきっかけに、ビルド・Lint・テストを自動実行して、変更の安全性を確認する」  
CD「CDは、その確認を通った成果物を staging や production に反映して、使える状態まで運ぶ」  
CI「GitHub Actions は、その一連の流れを workflow として書ける」  
CD「job を分けて、needs でつないで、条件分岐もできる」  
CI「だから“Actions使ってる＝CI/CDやってる”ではなくて」  
CD「何を自動化してるかで、CIだけなのか、CDまで入ってるのかが変わる」  
CI「ええやん、今日はちゃんと技術寄りや」  
CD「たまにはな」  

CI「じゃあ最後、お前の理想言うてみ」  
CD「はい。PR作成でCIが走る」  
CI「うん」  
CD「レビュー通過後にmainへマージ」  
CI「うんうん」  
CD「その後GitHub Actionsで自動デプロイ」  
CI「うん」  
CD「そして障害ゼロ！」  
CI「それは願望や」  
  
CD「せめて最後くらい夢見させてえな！」  
  
二人「どうも、ありがとうございましたー！」
