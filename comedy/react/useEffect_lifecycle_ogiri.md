### おまけ：この大喜利でうっすら伝えたいこと

* **ライフサイクル**
  コンポーネントが
  「出てくる（マウント）」
  「変わる（アップデート）」
  「消える（アンマウント）」
  の流れ
* **useEffect**
  その流れの中で、**画面表示そのものとは別にやりたい処理**を動かすもの
* **cleanup**
  付けたイベントやタイマーなどを、**消える前や再実行前に片付ける処理**
* **依存配列**
  「どの値が変わったときに effect を走らせるか」の指定
* **無限ループ注意**
  effect の中で state を更新して、その更新条件が雑だとぐるぐる回る

必要なら次に、同じノリで
**「依存配列あり・なし・空配列の違い大喜利」**
か
**「クラスコンポーネントの lifecycle メソッド vs useEffect 大喜利」**
の形で増やせます。
