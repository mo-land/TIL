## 理解したこと
- ディレクトリ設計の重要性
- ディレクトリ設計パターンは、1つではない。
  - プロジェクトの規模やチームの運用方法に応じて柔軟に選択・適用すると良い
- 主要なディレクトリ設計パターン
  - By File Type （タイプ別）
    - ファイルの種類ごとにフォルダを分ける
  - Feature-Based（フィーチャーベース）
    - アプリの機能ごとにフォルダを作成し、その中に関連するすべてのコードを1フォルダにまとめて管理する
  - Atomic Design（アトミックデザイン）
    - カリキュラムの説明を参照。
- デザインパターン  
  → 開発でよく発生する課題に対して、過去の経験や成功事例をもとに作成されたパターン  
  - Container/Presentationalパターン  
    → 仕事を分担するのが得意  
    → 「ロジック担当」と「見た目担当」を分けたい時向き  
  - HOC  
    → 同じ機能をまとめて足すのが得意  
    → 認証、計測、権限チェックみたいな横断処理向き  
  - Render Props  
    → ロジックを渡して表示は自由にしたい時に強い  
    → 柔軟だけど、書き方が込み入ることがある
